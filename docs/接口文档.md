# AIGC 智能短视频创作助手接口文档

## 1. 文档说明

本文档描述《AIGC 智能短视频创作助手》后端 REST API 的调用方式、请求参数、响应结构和常见错误。

- 后端基础地址：`http://localhost:3000`
- API 基础前缀：`/api`
- 请求格式：`application/json`
- 响应格式：`application/json`
- 鉴权方式：JWT Bearer Token

## 2. 通用约定

### 2.1 成功响应

```json
{
  "success": true,
  "data": {}
}
```

部分接口会额外返回 `message`、`token`、`user`、`generationMode`、`fallbackReason` 等字段。

### 2.2 失败响应

```json
{
  "success": false,
  "message": "错误信息"
}
```

### 2.3 鉴权请求头

除注册、登录、忘记密码、健康检查外，其他业务接口均需要登录。

```http
Authorization: Bearer <JWT_TOKEN>
```

### 2.4 常见状态码

| 状态码 | 说明 |
| --- | --- |
| `200` | 请求成功 |
| `400` | 请求参数错误 |
| `401` | 未登录、Token 缺失、Token 无效或过期 |
| `404` | 数据不存在或无权限访问 |
| `409` | 用户名或邮箱等唯一字段冲突 |
| `500` | 服务器内部错误或数据库连接异常 |

## 3. 健康检查接口

### 3.1 健康检查

- 请求方式：`GET`
- 请求路径：`/api/health`
- 是否需要登录：否

#### 响应示例

```json
{
  "success": true,
  "message": "AIGC short video assistant API is running"
}
```

## 4. 用户认证模块

### 4.1 用户注册

- 请求方式：`POST`
- 请求路径：`/api/auth/register`
- 是否需要登录：否

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | `string` | 是 | 用户名，不能重复 |
| `email` | `string` | 是 | 邮箱，不能重复，需符合邮箱格式 |
| `password` | `string` | 是 | 密码，8-20 位，必须包含字母和数字，不能包含空格 |

#### 请求示例

```json
{
  "username": "vicky",
  "email": "vicky@example.com",
  "password": "abc123456"
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "注册成功",
  "user": {
    "id": 1,
    "username": "vicky",
    "email": "vicky@example.com"
  }
}
```

#### 常见错误

| 场景 | 返回说明 |
| --- | --- |
| 用户名、邮箱或密码为空 | `400` |
| 邮箱格式错误 | `400` |
| 密码不符合规则 | `400` |
| 用户名已存在 | `409` |
| 邮箱已注册 | `409` |

### 4.2 用户登录

- 请求方式：`POST`
- 请求路径：`/api/auth/login`
- 是否需要登录：否

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `email` | `string` | 是 | 注册邮箱 |
| `password` | `string` | 是 | 用户密码 |

#### 请求示例

```json
{
  "email": "vicky@example.com",
  "password": "abc123456"
}
```

#### 响应示例

```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6...",
  "user": {
    "id": 1,
    "username": "vicky",
    "email": "vicky@example.com"
  }
}
```

#### 说明

- Token 有效期由 `JWT_EXPIRES_IN` 控制，默认 `7d`。
- 前端登录成功后会将 token 保存到 `localStorage`，后续请求自动携带。

- 请求路径：`/api/auth/profile`
- 是否需要登录：是

#### 请求头


#### 响应示例

```json
    "email": "vicky@example.com",

```

### 4.4 修改用户信息

- 请求方式：`PUT`
- 请求路径：`/api/auth/profile`
- 是否需要登录：是

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | `string` | 是 | 新用户名，不能与其他用户重复 |

#### 请求示例

```json
{
  "username": "vicky_new"
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "修改成功",
  "data": {
    "id": 1,
    "username": "vicky_new",
    "email": "vicky@example.com"
  }

```

### 4.5 修改密码

- 请求方式：`PUT`
- 请求路径：`/api/auth/password`
- 是否需要登录：是

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `oldPassword` | `string` | 是 | 原密码 |
| `newPassword` | `string` | 是 | 新密码，需满足密码规则 |

#### 请求示例

```json
{
  "oldPassword": "abc123456",
  "newPassword": "newabc123456"
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "密码修改成功"
}
```

### 4.6 请求密码重置验证码

- 请求方式：`POST`
- 请求路径：`/api/auth/password-reset/request`
- 是否需要登录：否

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `email` | `string` | 是 | 注册邮箱 |

#### 请求示例

```json
{
  "email": "vicky@example.com"
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "如果邮箱已注册，验证码将发送到该邮箱"
}
```

#### 说明

- 为避免泄露账号是否存在，即使邮箱未注册，也返回统一成功提示。
- 验证码有效期为 10 分钟。
- 验证码以哈希形式存储在 `password_reset_codes` 表中。

### 4.7 确认重置密码

- 请求方式：`POST`
- 请求路径：`/api/auth/password-reset/confirm`
- 是否需要登录：否

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `email` | `string` | 是 | 注册邮箱 |
| `code` | `string` | 是 | 6 位邮箱验证码 |
| `newPassword` | `string` | 是 | 新密码，需满足密码规则 |

#### 请求示例

```json
{
  "email": "vicky@example.com",
  "code": "123456",
  "newPassword": "newabc123456"
}
```

#### 响应示例

```json
{
  "success": true,
  "message": "密码重置成功，请重新登录"
}
```

## 5. 创作生成模块

### 5.1 生成短视频创作方案

- 请求路径：`/api/creation/generate`
- 是否需要登录：是

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `topic` | `string` | 是 | 视频主题 |
| `platform` | `string` | 是 | 发布平台，可选：`抖音`、`小红书`、`B站`、`视频号` |
| `style` | `string` | 是 | 内容风格 |
| `duration` | `number` | 是 | 视频时长，可选：`15`、`30`、`60`、`90` |
| `audience` | `string` | 是 | 目标受众 |
| `creativeRequirement` | `string` | 否 | 自由创作要求，最多保留 500 字 |
| `generationMode` | `string` | 否 | 生成模式，可选：`template`、`ai` |
| `generationModePreference` | `string` | 否 | 生成模式偏好，可选：`template`、`ai` |


## 9. Postman 调用建议

### 9.1 登录并保存 Token

1. 调用 `POST /api/auth/login`。
2. 从响应中复制 `token`。
3. 在 Postman 的 Authorization 中选择 `Bearer Token`。
4. 粘贴 token 后即可调用需要登录的接口。

### 9.2 推荐测试顺序

1. `GET /api/health`
2. `POST /api/auth/register`
3. `POST /api/auth/login`
4. `GET /api/auth/profile`
5. `POST /api/creation/generate`
6. `POST /api/creation/save`
7. `GET /api/creation/history`
8. `GET /api/creation/detail/:id`
9. `GET /api/hot-topics`
10. `DELETE /api/creation/:id`

## 10. 环境变量说明

后端环境变量位于 `server/.env`，示例见 `server/.env.example`。

| 变量名 | 说明 |
| --- | --- |
| `PORT` | 后端端口，默认 `3000` |
| `DB_HOST` | MySQL 地址 |
| `DB_PORT` | MySQL 端口 |
| `DB_USER` | MySQL 用户名 |
| `DB_PASSWORD` | MySQL 密码 |
| `DB_NAME` | 数据库名称 |
| `JWT_SECRET` | JWT 签名密钥，生产环境必须修改 |
| `JWT_EXPIRES_IN` | Token 有效期，默认 `7d` |
| `AI_PROVIDER` | AI 服务提供商，当前支持 DeepSeek 配置 |
| `AI_API_KEY` | AI API Key，不允许提交到 GitHub |
| `AI_BASE_URL` | AI 服务地址，例如 `https://api.deepseek.com` |
| `AI_MODEL` | AI 模型名称，例如 `deepseek-chat` |
| `GENERATION_MODE` | 默认生成模式，可选 `template`、`ai` |
| `SMTP_HOST` | 邮箱 SMTP 地址 |
| `SMTP_PORT` | 邮箱 SMTP 端口 |
| `SMTP_SECURE` | 是否使用安全连接 |
| `SMTP_USER` | 邮箱账号 |
| `SMTP_PASS` | 邮箱授权码或密码 |
| `SMTP_FROM` | 发件人邮箱 |

## 11. 安全说明

- 密码使用 bcrypt 加密保存。
- JWT 仅用于接口鉴权，不应长期暴露在 URL 中。
- 所有需要用户身份的接口均通过 `authMiddleware` 校验。
- 用户只能查看和删除自己的历史记录。
- 数据库操作使用参数化查询，降低 SQL 注入风险。
- `.env` 不应提交到 GitHub。
- AI API Key 只应保存在服务器环境变量中。
