# AIGC 智能短视频创作助手 UI 自动化测试用例

## 1. 文档说明

本文档用于后续编写 UI 自动化测试代码，推荐使用 Playwright。用例按照页面和核心业务流程拆分，覆盖登录注册、路由鉴权、工作台、智能创作、历史记录、个人中心、多语言和异常场景。

## 3. 自动化测试通用约定

### 3.1 测试账号

建议自动化脚本使用独立测试账号，避免污染真实演示数据。

```text
用户名：uitest_user
邮箱：uitest_user@example.com
密码：Test123456
```

### 3.2 数据准备策略

自动化测试可以采用两种方式准备登录态：

1. 通过 UI 注册/登录获取真实 token。
2. 通过接口登录后写入 `localStorage`，减少重复登录成本。

推荐公共登录态：

```js
await page.evaluate(({ token, user }) => {
  localStorage.setItem('token', token)
  localStorage.setItem('user', JSON.stringify(user))
}, { token, user })
```

### 3.3 建议选择器策略

当前项目已为登录、注册、导航、创作、历史记录和个人中心等核心控件补充 `data-testid`，编写自动化代码时建议：

- 优先使用角色选择器：`getByRole`
- 其次使用文本：`getByText`
- 表单输入可使用 label 或 placeholder
- Element Plus 下拉框需要点击后选择弹出的 `.el-select-dropdown__item`
- 优先使用已有 `data-testid` 定位关键按钮、输入框和表格

### 3.4 通用断言点

| 类型 | 断言 |
| --- | --- |
| 页面跳转 | `expect(page).toHaveURL(...)` |
| 文本展示 | `expect(locator).toBeVisible()` |
| 表单校验 | 页面出现错误提示 |
| 接口调用 | `page.waitForResponse(...)` |
| 本地存储 | `localStorage.token`、`localStorage.user` |
| Toast 提示 | `.el-message` 出现对应提示 |
| 弹窗/抽屉 | `.el-dialog`、`.el-drawer` 可见 |

## 18. 后续生成代码时的注意事项

- 优先使用已补充的 `data-testid`，Element Plus 下拉框、弹窗等浮层组件可结合角色选择器或可见文本定位。
- 自动化测试运行前应保证后端服务、前端服务和 MySQL 都已启动。
- 涉及邮箱验证码的用例建议先用 mock 或后端测试模式处理，不建议依赖真实邮箱收件。
- 生成类接口耗时不稳定，Playwright 等待时间需要适当放宽。
- 历史记录删除和清空会改变数据，建议使用专用测试账号。
- AI 模式测试不应强依赖真实 API Key，可通过接口 mock 验证 UI 展示。
